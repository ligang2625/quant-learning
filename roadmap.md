# 低频量化选股学习路线

## 1. 个人背景与学习目标

### 当前基础

- 已学习基础 Python 编程。
- 已学习基础机器学习。
- 有约一年的股票投资经验。
- 当前短板主要是金融知识、量化回测、因子研究和交易系统化能力。

### 学习目标

建立一套低频量化分析程序，用于辅助：

- 股票筛选；
- 买入候选判断；
- 卖出或减仓信号判断；
- 策略回测；
- 风险控制；
- 定期生成选股和策略报告。

目标不是一开始就预测股价，而是逐步搭建一个完整的研究闭环：

```text
数据获取 → 数据清洗 → 因子计算 → 选股排序 → 买卖信号 → 回测验证 → 风控约束 → 报告输出
```

## 5. 第三阶段：技术择时与简单回测

这一阶段先不要急着做复杂选股，而是先理解买卖信号和回测流程。

### 5.1 第一个择时策略

可以从均线策略开始：

```text
20 日均线上穿 60 日均线买入；
跌破 60 日均线卖出；
加入成交额过滤；
加入止损或移动止盈；
和基准指数对比。
```

### 5.2 回测中必须注意的问题

重点理解：

- 信号日和成交日不能混淆；
- 不要使用未来数据；
- 要扣除手续费；
- 要考虑滑点；
- 要考虑涨跌停无法成交；
- 要考虑停牌；
- 要输出完整交易记录；
- 不能只看收益率，还要看回撤。

### 5.3 必须输出的回测指标

包括：

- 总收益；
- 年化收益；
- 最大回撤；
- 夏普比率；
- Calmar 比率；
- 胜率；
- 盈亏比；
- 换手率；
- 年度收益；
- 月度收益；
- 相对基准超额收益。

## 6. 第四阶段：因子选股

这是低频量化选股的核心。

### 6.1 常见因子类型

| 因子类型 | 示例 |
|---|---|
| 价值因子 | PE、PB、股息率、现金流收益率 |
| 质量因子 | ROE、毛利率、净利率、经营现金流/利润 |
| 成长因子 | 营收增速、利润增速、ROE 改善 |
| 动量因子 | 20 日收益率、60 日收益率、120 日收益率 |
| 风险因子 | 波动率、Beta、最大回撤 |
| 流动性因子 | 成交额、换手率、市值 |

### 6.2 因子研究流程

标准流程：

```text
获取数据
→ 计算因子
→ 去极值
→ 标准化
→ 行业/市值中性化
→ 计算 IC / Rank IC
→ 分组回测
→ 检查换手率
→ 检查不同市场环境下的稳定性
```

### 6.3 因子分析重点概念

必须掌握：

- z-score 标准化；
- winsorize 去极值；
- 行业中性化；
- 市值中性化；
- IC；
- Rank IC；
- 分组收益；
- 多空组合；
- 因子衰减；
- 因子换手率。

### 6.4 多因子打分

可以先采用简单加权：

```text
综合得分 = 质量得分 × 0.3 + 价值得分 × 0.3 + 动量得分 × 0.3 + 流动性得分 × 0.1
```

后续可以升级为：

- IC 加权；
- 回归模型；
- LightGBM / XGBoost 排序模型；
- 风险模型约束下的组合优化。

## 7. 第五阶段：组合构建与风险控制

很多初学者只关注选股，但真正影响长期表现的是组合和风控。

### 7.1 组合构建

可以从简单等权开始：

```text
每月调仓一次；
选择综合评分前 20 只股票；
每只股票等权配置 5%；
下个月重新计算评分并调仓。
```

后续可以加入：

- 行业权重限制；
- 单票仓位限制；
- 流动性限制；
- 市值暴露限制；
- 波动率倒数加权；
- 风险平价思想。

### 7.2 风控规则

建议加入：

- 单只股票最大仓位不超过 5%；
- 单一行业最大仓位不超过 25%；
- 剔除 ST 股票；
- 剔除长期停牌股票；
- 剔除成交额过低股票；
- 剔除财务异常股票；
- 大盘跌破长期均线时降低总仓位；
- 最大回撤超过阈值时降仓；
- 控制组合换手率。

### 7.3 重点风险指标

| 指标 | 作用 |
|---|---|
| 年化收益 | 衡量长期收益 |
| 最大回撤 | 衡量最差体验 |
| 夏普比率 | 衡量风险调整收益 |
| Calmar 比率 | 衡量收益和回撤关系 |
| 胜率 | 辅助观察交易质量 |
| 盈亏比 | 衡量赚钱交易和亏钱交易的比例 |
| 换手率 | 衡量交易成本压力 |
| 超额收益 | 衡量是否跑赢基准 |

## 8. 第六阶段：机器学习选股

你已经有机器学习基础，但在量化中要特别注意数据泄露和过拟合。

### 8.1 不建议一开始做的事情

不建议：

- 直接预测明天涨跌；
- 直接用深度学习预测股价；
- 随机划分训练集和测试集；
- 用未来财务数据；
- 过度调参；
- 只看回测收益，不看稳定性。

### 8.2 推荐的机器学习目标

更合适的目标是横截面排序，例如：

```text
预测未来 20 个交易日收益是否进入股票池前 30%；
预测未来一个月的相对收益排名；
预测股票未来一个月是否跑赢基准。
```

### 8.3 推荐模型顺序

从简单到复杂：

```text
线性回归 / 逻辑回归
随机森林
XGBoost / LightGBM
排序模型
神经网络
```

### 8.4 机器学习验证方式

必须使用：

- 时间序列切分；
- walk-forward validation；
- 滚动训练；
- 样本外测试；
- 不同市场阶段测试；
- 特征重要性分析；
- 模型稳定性检查。

## 9. 常用量化分析框架

### 9.1 推荐技术栈

最适合当前阶段的组合：

```text
数据：AKShare / Tushare
数据处理：pandas / numpy
回测：vectorbt / backtesting.py / 自写回测逻辑
报告：QuantStats
因子分析：参考 Alphalens 思路
机器学习进阶：Qlib
实盘进阶：vn.py
```

### 9.2 常用框架对比

| 类别 | 框架/平台 | 适合程度 | 说明 |
|---|---|---|---|
| 数据处理 | pandas / numpy | 必学 | 所有量化研究的基础 |
| 轻量回测 | backtesting.py | 适合入门 | 简单策略、单标的策略容易上手 |
| 向量化回测 | vectorbt | 很适合 | 适合多股票、多参数、低频研究 |
| 传统回测 | Backtrader | 可选 | 功能完整，但学习曲线略高 |
| 事件驱动 | Zipline-reloaded | 进阶 | 更接近机构化事件驱动回测 |
| 因子分析 | Alphalens | 建议学习思想 | 用于 IC、分组收益、因子分析 |
| 绩效报告 | QuantStats | 很适合 | 自动生成收益和风险报告 |
| 机器学习量化 | Qlib | 进阶推荐 | 适合机器学习选股研究 |
| 国内实盘 | vn.py / VeighNa | 后期学习 | 更偏实盘接口和交易系统 |
| 云平台 | 聚宽 / 米筐 / BigQuant | 可用于快速练习 | 适合快速体验 A 股策略研究 |

### 9.3 当前最推荐组合

当前阶段建议使用：

```text
pandas + numpy + AKShare/Tushare + vectorbt + QuantStats
```

原因：

- 学习曲线相对平滑；
- 适合低频选股；
- 适合批量测试股票和参数；
- 容易扩展机器学习模块；
- 适合自己搭建完整项目。

## 14. 学习重点提醒

### 14.1 最重要的不是机器学习

你已经有机器学习基础，但低频量化中更重要的是：

- 数据是否真实；
- 回测是否正确；
- 因子是否有经济逻辑；
- 组合是否能承受回撤；
- 策略是否稳定；
- 交易成本是否合理。

### 14.2 必须避免的错误

避免：

- 未来函数；
- 生存者偏差；
- 财报数据发布时间穿越；
- 忽略交易成本；
- 忽略滑点；
- 忽略涨跌停；
- 忽略停牌；
- 过度优化参数；
- 只看收益不看回撤；
- 用随机切分训练机器学习模型；
- 只在一段行情里验证策略。

### 14.3 更合理的目标

合理目标是：

```text
构建一个长期稳定、可解释、可回测、可监控的低频选股系统。
```

不合理目标是：

```text
准确预测每只股票明天涨跌；
每天抓涨停；
靠一个模型长期稳定暴利；
用机器学习自动发现稳赚策略。
```
